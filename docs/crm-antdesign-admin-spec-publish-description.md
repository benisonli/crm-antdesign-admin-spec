# crm-antdesign-admin-spec｜发布说明文案

## 一句话卖点
把“后台页面别乱做”这件事变成可复用规则：先判断页面类型，再按 Ant Design 中后台逻辑稳定生成 CRM / Admin 页面。

---

## 简介（短版）
`crm-antdesign-admin-spec` 是一个面向 B 端 CRM / Admin 场景的设计与生成规范 skill，帮助模型在做统计页、AI 分析页、列表页、配置页时保持结构稳定、风格统一、语义合理，并更贴近 Ant Design 的真实实现方式。

---

## 简介（长版）
很多 AI 在生成后台页面时有两个常见问题：

1. 把所有页面都做成“看板大卡片”
2. 为了好看牺牲信息密度、结构清晰度和工程可实现性

`crm-antdesign-admin-spec` 的目标，就是把这类问题前置约束掉。

它通过一套清晰的页面分类方法、视觉规则、Token 建议、组件映射和 Prompt 模板，让模型在面对 B 端 CRM / Admin 页面时，优先输出：

- 合理的页面类型判断
- 稳定的后台骨架
- 接近 Ant Design 的组件语义
- 可落地的设计方向
- 适合 UIUXProMax 或其他 AI 页面生成工具继续执行的输入规范

这个 skill 特别适合：
- CRM 系统
- SaaS 管理后台
- 数据统计页面
- AI 分析看板
- 客户列表页
- 配置/参数/工具页

---

## 它解决什么问题

### 1. 防止页面类型误判
不是每个页面都该长成 dashboard。

这个 skill 会先帮助模型判断：
- 统计页
- AI 分析页
- 列表页
- 配置/工具页

再决定页面骨架和组件结构。

### 2. 防止后台页面“官网化”
它会明确约束模型避免：
- 营销官网风
- 玻璃拟态泛滥
- 赛博朋克/发光边框
- 移动端化后台页面

### 3. 提高实现友好度
它不是纯视觉灵感 skill，而是偏工程可落地：
- Ant Design 风格语义
- Token 指引
- 组件映射
- 页面骨架模板

### 4. 提高 AI 页面生成质量
当它与 UIUXProMax 或其他生成能力结合时，可以显著降低：
- 页面结构漂移
- 风格不一致
- 组件不合理
- 页面看起来“很炫但不能做”

---

## 适合谁
- 产品经理
- UI / UX 设计师
- 前端工程师
- 用 AI 生成后台页面的人
- 维护 CRM / SaaS Admin 设计一致性的人

---

## 典型使用场景
- 让模型先定义一个 AI 客户洞察页的页面结构
- 让模型生成一个 Ant Design 风格的客户列表页
- 评审一个配置页是否被错误做成看板风
- 给 UIUXProMax 准备更稳定的后台页面 Prompt
- 统一一个团队对 CRM/Admin 页面类型的理解

---

## 输出结果通常包括
- 页面类型判断
- 页面目标和信息架构
- 模块/区块拆分
- Ant Design 组件映射
- 视觉约束与禁止项
- 可直接继续喂给 UIUXProMax 的 Prompt 模板

---

## 不适合的场景
这个 skill 不适合：
- 官网
- Landing Page
- 偏品牌表达的视觉页
- C 端 App 设计
- 以艺术感优先于实施性的页面
